# Butemine School Website — CLAUDE.md

## Project Overview

Official website for **Butemine English Medium Pre & Primary School**, Samunge Village, Ngorongoro District, Tanzania. Founded 2013 by John Nedura & Grace Ginduri.

Stack: Node.js · Express.js · EJS templates · Tailwind CSS v3 · vanilla JS.

---

## Architecture

```
server.js           ← All routes defined inline (no routes/ folder)
views/
  partials/
    head.ejs        ← <head>, fonts, CSS link, dark-mode init script
    nav.ejs         ← Fixed navbar, mobile menu, dark/light toggle
    footer.ejs      ← Footer, WhatsApp FAB, back-to-top button, main.js
  index.ejs         ← Home (8 sections)
  about.ejs
  admissions.ejs
  schools.ejs
  gallery.ejs
  news.ejs
  results.ejs
  contact.ejs
  donate.ejs
  404.ejs
src/input.css       ← Tailwind directives + custom CSS classes
public/
  css/style.css     ← Generated by Tailwind (do not edit)
  js/main.js        ← Scroll effects, mobile menu, lightbox, counters, dark mode
tailwind.config.js
server.js
```

## Routing

All routes live in `server.js`. Each passes a `page` string used by nav.ejs to highlight the active link.

| URL           | View        | `page` value  |
|---------------|-------------|---------------|
| `/`           | index       | `'home'`      |
| `/about`      | about       | `'about'`     |
| `/admissions` | admissions  | `'admissions'`|
| `/schools`    | schools     | `'schools'`   |
| `/gallery`    | gallery     | `'gallery'`   |
| `/news`       | news        | `'news'`      |
| `/results`    | results     | `'results'`   |
| `/contact`    | contact     | `'contact'`   |
| `/donate`     | donate      | `'donate'`    |
| `*`           | 404         | `''`          |

`/logo.png` is served from the root-level file `Butemine Logo.png` via a dedicated route (not from `public/`).

## EJS Layout Pattern

No layout engine — each view manually includes partials:

```ejs
<%- include('partials/head', { title: 'Page Title' }) %>
<body class="font-body bg-cream overflow-x-hidden">
<%- include('partials/nav') %>

<!-- page content -->

<%- include('partials/footer') %>
</body>
</html>
```

Variables passed to partials:
- `title` → inserted into `<title>` tag in head.ejs (`"<title> | Butemine School — Tanzania"`)
- `page` → available in nav.ejs via closure (passed to `res.render`)

## Styling Conventions

### Color Palette (tailwind.config.js)

> **Warning:** The color aliases are named after texture/material, not hue. The actual colors differ from the names:

| Alias        | Actual colour             | Usage                          |
|--------------|---------------------------|--------------------------------|
| `terracotta` | Cornflower **blue** scale | Primary accent (logo globe)    |
| `gold`       | **Green** scale           | Secondary accent (logo ribbon) |
| `forest`     | Same green as `gold`      | Duplicate — prefer `gold`      |
| `bark`       | Deep **navy** `#0D1B3E`   | Dark backgrounds, text         |
| `cream`      | `#F4F8FF` (blue-tinted)   | Light page background          |

### Fonts

- `font-display` → Playfair Display (headings, quotes)
- `font-body` → DM Sans (body text, UI)

### Reusable CSS Classes (defined in `src/input.css`)

| Class             | Purpose                                      |
|-------------------|----------------------------------------------|
| `.btn-primary`    | Main CTA button (blue)                       |
| `.btn-outline`    | Outline button (white border, dark bg)       |
| `.btn-outline-dark` | Outline button for light backgrounds       |
| `.btn-gold`       | Gold/green filled CTA                        |
| `.btn-donate-nav` | Nav-specific donate button variant           |
| `.card`           | White card with shadow, hover lift           |
| `.nav-link`       | Nav anchor with underline-on-hover effect    |
| `.reveal`         | Scroll-reveal target (toggled by JS observer)|
| `.section-label`  | Small uppercase eyebrow text                 |
| `.section-title`  | Large display heading                        |
| `.section-title-light` | section-title for dark backgrounds      |
| `.gold-divider`   | Short decorative gold/green horizontal rule  |
| `.hero-pattern`   | Dot-grid SVG background texture              |
| `.gallery-item`   | Gallery grid cell with overlay               |
| `.overlay`        | Dark gradient overlay on gallery images      |
| `.testimonial-card` | Positioned testimonial slide               |
| `.whatsapp-btn`   | WhatsApp floating button                     |

### Dark Mode

Strategy: `darkMode: 'class'` in Tailwind. Toggle via `document.documentElement.classList.add('dark')`. Theme persisted to `localStorage`. System preference is the default for first-time visitors. Anti-flash script runs in `<head>` before render.

## Reusable UI Sections (Home page reference)

All major sections of `index.ejs` are templates for reuse across pages:

1. **Hero** — full-viewport gradient, floating orbs, wave SVG divider
2. **Stats** — animated counters with `data-target` attributes
3. **Services / Program cards** — `.card` grid, EJS `forEach` loop
4. **About preview** — 2-col image + text, gold blockquote
5. **Gallery grid** — 3-col CSS grid with lightbox (`main.js`)
6. **Testimonials** — slide carousel, `.testimonial-card` + `.t-active`
7. **Location** — green gradient section with nearby attractions grid
8. **Community Impact** — terracotta section, stat tiles
9. **Contact CTA** — card with decorative right panel

## Deployment Notes

- No build step beyond `npm run build:css` — commit `public/css/style.css`
- Set `NODE_ENV=production` and `PORT` env vars on host
- Consider a reverse proxy (nginx/Caddy) for HTTPS, gzip, and static asset caching
- No database — all content is hard-coded in EJS templates
- Domain: `www.butemine.ac.tz`

## Suggested Improvements

### SEO
- **Per-page meta descriptions** — pass a `description` variable from each route and render it in `head.ejs` instead of the static string
- **Open Graph tags** — add `og:title`, `og:description`, `og:image`, `og:url` in `head.ejs`
- **Structured data** — add `<script type="application/ld+json">` with `School` or `Organization` schema on the home page
- **Canonical tag** — `<link rel="canonical" href="...">` per page
- **`sitemap.xml`** — generate statically and serve from `public/`
- **`robots.txt`** — add to `public/`
- **Twitter Card meta** — `twitter:card`, `twitter:title`, `twitter:description`

### Performance
- **Compress responses** — add `compression` npm package as Express middleware
- **Font subsetting** — add `&subset=latin` to Google Fonts URL; consider self-hosting with `@font-face` to eliminate the render-blocking request
- **Image lazy loading** — add `loading="lazy"` to all below-fold `<img>` tags
- **Static asset cache headers** — set long `Cache-Control` on `/css/`, `/js/`, `/images/` in Express or via reverse proxy
- **WebP images** — convert `File N.jpg` to WebP and use `<picture>` with JPEG fallback
- **Logo filename** — rename `Butemine Logo.png` → `logo.png` in the project root to remove the space; update the route in `server.js`

### Accessibility
- **Skip-to-content link** — add `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to content</a>` at the top of `nav.ejs`
- **`<main>` landmark** — wrap page body content in `<main id="main-content">` in each view
- **Gallery alt text** — use descriptive `alt` per image instead of generic `"Butemine photo"`
- **Testimonial carousel** — add `role="region"` and `aria-label="Testimonials"` to the carousel wrapper; add `aria-live="polite"` for slide changes
- **Color contrast audit** — `text-white/50` and `text-bark/60` overlaid on light backgrounds may fall below WCAG AA; verify with a contrast checker

### Mobile Responsiveness
- **Hero `<h1>` font size** — `text-7xl` at md breakpoint is ~72px; test on 360px-wide devices for overflow
- **Gallery grid** — the 3-column grid (`grid-cols-3`) is used at all sizes; consider `grid-cols-2` below `sm` breakpoint
- **Footer 4-column grid** — already uses `grid-cols-1 md:grid-cols-2 lg:grid-cols-4`; verify the 2-col mid-state on tablets (768px)
- **Touch targets** — hamburger button spans are `w-6 h-0.5`; the `p-2` wrapper provides ~40px tap target — acceptable but verify on device
